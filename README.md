# Zapret UI

**Графическая оболочка для движка обхода DPI [zapret2](https://github.com/bol-van/zapret2) (winws2) под Windows. [Телегам канал проекта](https://t.me/Zapret2UI)**

**[Комьюнити чат, Changlog, новости по проекту в телеграм канале или же в поиске Zapret2UI](https://t.me/Zapret2UI)**

Скачивает официальный движок с GitHub, проверяет контрольные суммы, держит его в актуальном
состоянии — и даёт обходить блокировки одной кнопкой, без правки `.cmd`-файлов вручную.

## Что это

Это **не VPN и не прокси**. Когда вы открываете сайт, ваш компьютер шлёт серверу первый пакет, где
**открытым текстом видно имя сайта** (SNI в HTTPS, `Host:` в HTTP). Провайдерский DPI читает это поле и
блокирует или замедляет соединение. Движок **winws2** стоит между приложением и сетью и **портит/разбивает
именно этот первый пакет** так, что DPI его не распознаёт, а настоящий сервер — понимает.

Итог: блокировку **по имени сайта** обход снимает, блокировку **по IP** — нет (для неё нужен VPN).

**Zapret UI** даёт этому движку удобную обёртку: старт/стоп одной кнопкой, автоподбор и генерацию рабочей
стратегии, готовые пресеты, списки доменов, живой лог, автозапуск и тихое автообновление.

> ⚠️ Инструмент для исследовательских и образовательных целей, восстановления доступа к легальным ресурсам
> и тестирования сетей. Используйте по законам своей страны.

## 🚀 Быстрый старт

1. Запустите **от администратора** (без этого WinDivert не загрузит драйвер — будет запрос UAC).
2. Нажмите **«Включить обход»**. Применится рекомендованный пресет, движок запустится, статус станет
   зелёным.
3. Проверьте нужный сайт.

**Не помогло?** Это нормально — обход зависит от провайдера, универсального пресета нет. Дальше:

- Нажмите **«Переподобрать под провайдера»** прямо на главной — приложение само найдёт рабочую стратегию.
- Если и это не пробило — откройте **[подбор и генерацию стратегий](#-подбор-и-генерация)**
  (вкладка **Диагностика** в расширенном режиме). Это правильный путь, когда готовые пресеты не подошли.

Переключатель **Простой / Расширенный** — вверху по центру окна. Простой — одна кнопка; расширенный —
вкладки с полным управлением.

## Интерфейс — куда жать

Шесть вкладок в расширенном режиме: **Главная · Стратегии · Хостлисты · Диагностика · Журнал · Настройки**.

### Главная

<img src="docs/home-advanced.png" width="720" alt="Главная"/>

- **Включить / Выключить обход** — большая кнопка по центру: старт/стоп движка.
- **Что подобрать** — `Discord + YouTube` / `Discord` / `YouTube`: цель для автоподбора.
- **Переподобрать под провайдера** — запускает автоподбор по выбранной цели.
- В простом режиме слева — карточка **Telegram** (доступ через ваш MTProxy в один клик) и QR поддержки.

### Стратегии

<img src="docs/strategies.png" width="720" alt="Стратегии"/>

Готовые «комбо»-пресеты. Каждый в одной команде разводит трафик по SNI (Discord / YouTube / остальное →
разные десинки) и несёт профиль голоса Discord — переключаться между пресетами не нужно. Группы:

- **Основные (Discord / YouTube)** — `Комбо (рекомендуемый)`, `Комбо — Flowseal (multisplit seqovl)`,
  `Комбо — Flowseal ALT (fake+fakedsplit)`, `Комбо — окно (wssize)`, `Discord — голос (QUIC-фейк)`.
- **Telegram (через ваш прокси)** — 3 варианта под ваш MTProxy (см. [Telegram](#telegram)).
- **✨ Сгенерировано автоподбором** и **Личные** — ваши сохранённые и копии.

Куда жать: выберите пресет → **Запустить** (в шапке). Работающий помечен значком **ВКЛЮЧЁН**.
**Аргументы движка** — показать/править реальную командную строку. **Дублировать** — сделать свою копию.
**Как это работает** — краткий экскурс прямо в приложении.

> Сменили пресет на ходу? Стратегию нельзя поменять без перезапуска движка, поэтому появится плашка
> **«Сменить стратегию»** — смена применится после подтверждения.

### Хостлисты

<img src="docs/hostlists.png" width="720" alt="Хостлисты"/>

Списки доменов, к которым применяется обход (один домен на строку). Встроенные
(`youtube`/`discord`/`telegram`/`exclude`/`general`) **пере-засеваются из приложения при каждом запуске**,
чтобы обновления доменов доходили до старых установок. Ваши списки и список **`proxy`** (адрес вашего
MTProxy) не трогаются.

Куда жать: список слева → правка справа → **Сохранить список**. **+ Новый список** — свой.
Хостлисты нужны не всегда: готовые комбо уже разводят трафик через `youtube`/`discord`. Правьте, только
если добавляете/убираете домены.

### Диагностика

<img src="docs/diagnostics.png" width="760" alt="Диагностика"/>

Слева — инструменты, справа — **матрица доступности**, которая масштабируется по ширине окна:

- **Подобрать лучшую** — автоподбор стратегии под вашего провайдера ([подробнее](#-подбор-и-генерация)).
- **✨ Сгенерировать стратегию** — собрать персональную стратегию под вашу сеть ([подробнее](#-подбор-и-генерация)).
- **Диагностика** — прогнать матрицу: видно, что именно блокируется (HTTP / TLS 1.2 / TLS 1.3 / пинг) по
  Discord, YouTube, Telegram, Cloudflare, Twitch, Google и DNS.
- **Свои цели** — добавить свой сайт: приложение найдёт его поддомены и связанные домены, и учтёт их при
  подборе, генерации и диагностике.

### Журнал

Живой вывод движка (`--debug`) + запись в файл. Сюда смотреть, **если движок не стартовал** — там будет
причина (ошибка аргументов, драйвер, антивирус).

### Настройки

<img src="docs/settings.png" width="760" alt="Настройки"/>

- **Движок** — версия + **Проверить обновления**; тумблер автообновления при запуске.
- **Запуск** — автозапуск при входе в Windows, сразу включать обход, сворачивать в трей вместо закрытия.
- **Авто-починка** — фоном следит за Discord/YouTube и, если обход отвалился, **молча переподбирает**
  рабочую стратегию.
- **Область обхода** — обходить **только списки** (по умолчанию, как Flowseal) или **все сайты**. См.
  [ниже](#область-обхода).
- **Игровой фильтр** — по умолчанию ВЫКЛ: обход не трогает игровой трафик. Включайте, только если душат
  саму игру.

## ✨ Подбор и генерация

*Когда готовые пресеты не подошли.* Обход **зависит от провайдера** — у разных операторов работает разное. Если кнопка «Включить обход» и
готовые пресеты не пробили, не перебирайте их вручную до бесконечности — пусть приложение подберёт само.
Оба инструмента — на вкладке **Диагностика**, оба требуют прав администратора (запускают движок) и
показывают живой прогресс в всплывающем окне.

| | **Подобрать лучшую** (автоподбор) | **✨ Сгенерировать стратегию** (генерация) |
|---|---|---|
| Что делает | Перебирает **встроенный набор** проверенных стратегий | **Собирает новую** стратегию из кирпичиков под вашу сеть |
| Как выбирает | Берёт ту, что пробила больше всего целей | Тестирует десинки **по каждому сервису** и склеивает лучший Discord + лучший YouTube |
| Результат | Применить / сохранить любую рабочую | Сохраняется как пресет **`✨ Сгенерировано …`** |
| Когда | Первый выбор — быстрее | Если автоподбор не нашёл — тоньше, но дольше |

**Как пользоваться:**

1. Вкладка **Диагностика** → выберите цель (`Discord + YouTube` / `Discord` / `YouTube`).
2. Нажмите **«Подобрать лучшую»** (или **«✨ Сгенерировать стратегию»**).
3. В окне прогресса видно, какие цели по каждой стратегии проходят (TLS 1.2 / 1.3).
4. По завершении — примените рабочую или сохраните в пресеты (можно несколько лучших).

> Telegram автоподбором/генерацией **не проверяется**: TLS-проба не отражает работу нативного клиента —
> для него отдельные прокси-пресеты (см. [Telegram](#telegram)).

## Область обхода

*Почему игра или приложение может «сломаться».* В **Настройках → Область обхода** — тумблер
**«Обходить все сайты, а не только из списков»**:

- **ВЫКЛ** *(по умолчанию, как у Flowseal)* — обход применяется лишь к YouTube, Discord, Telegram
  и вашим целям/хостлистам. Весь остальной трафик идёт нетронутым, поэтому **игры и приложения не ломаются**.
- **ВКЛ** — обход применяется ко всему TLS/QUIC (кроме списка исключений). Удобно — любой
  заблокированный сайт обходится сам, но может **сломать игру или приложение**, которого нет в исключениях
  (типичный симптом — бесконечный вход в аккаунт).

Если после включения тумблера перестала пускать игра — выключите его обратно и добавьте нужный
заблокированный сайт через **Свои цели** или хостлист.

## Если не работает

1. **Журнал.** Если движок не стартовал — там причина (аргументы, драйвер).
2. **Антивирус.** WinDivert — частая цель антивирусов; Defender может забрать `winws2.exe` /
   `WinDivert64.sys` в карантин. Добавьте папку `%LOCALAPPDATA%\ZapretUI\engine` в исключения.
3. **Диагностика.** Вкладка **Диагностика** → **Диагностика**: матрица покажет, что именно не открывается.
4. **Подбор/генерация.** Там же → **[Подобрать лучшую / Сгенерировать](#-подбор-и-генерация)**.
5. **Перебор вручную.** `Комбо (рекомендуемый)` → `Комбо — Flowseal` → `Комбо — Flowseal ALT`. Упирается
   вход в Discord — `Комбо — окно (wssize)`. Голос «подключается, но не слышно / NO_ROUTE» —
   `Discord — голос (QUIC-фейк)`.
6. **Блокировка по IP.** Если ресурс режут по IP (а не по имени) — обход бессилен, нужен VPN
   (см. [makeitfree.online](#vpn-от-автора--makeitfreeonline)).

### Полное удаление движка

Хотите удалить папку `engine`, а Windows пишет **«Файл уже используется»** на `WinDivert64.sys`? Это
драйвер ядра — он остаётся загруженным, пока его не выгрузить.

1. Полностью выйдите из приложения: **трей → Выход** (крестик только сворачивает в трей). Убедитесь, что в
   Диспетчере задач нет `winws2.exe`.
2. Откройте **PowerShell от администратора** и выгрузите драйвер:
   ```powershell
   sc.exe stop WinDivert
   sc.exe delete WinDivert
   ```
   > ⚠️ Именно **`sc.exe`**, а не `sc`! В PowerShell `sc` — это псевдоним команды `Set-Content`, поэтому
   > `sc stop WinDivert` молча не сработает. Дописывайте `.exe`. (В классической `cmd.exe` работает и `sc`.)
3. Теперь папка `engine` удалится.

**Проще:** просто **перезагрузите ПК** и удалите папку — драйвер настроен не грузиться сам, после
перезагрузки файл свободен.

## Telegram

Telegram — особый случай. Прямое (без прокси) подключение **приложения** Telegram к серверам в РФ во многих
сетях **десинком не восстанавливается** — по словам автора движка, это зависит от регулятора. Надёжный путь
для приложения — **MTProxy** (встроенный в Telegram прокси), а роль этой оболочки — **помочь клиенту
стабильно дозвониться до прокси**.

Пресеты **«Комбо + Telegram через ваш прокси»**:

- десинкуют рукопожатие **строго к IP вашего прокси** (по списку `proxy`); YouTube/Discord/голос и остальное
  не затрагиваются;
- заодно чинят **веб-версию** Telegram в браузере (`web.telegram.org`, `t.me`).

Вариантов **три** — отличаются только десинком прокси-рукопожатия; на разных операторах заходит разный,
пробуйте.

**Как включить:**

1. **Простой режим:** карточка **Telegram** → **«Включить для Telegram»**. **Расширенный:** вкладка
   **Стратегии** → группа **Telegram (через ваш прокси)** → вариант.
2. Введите **«Хост» из настроек вашего прокси в Telegram** (Настройки → Данные и память → Прокси → ваш
   прокси → «Хост»). Менять позже — вкладка **Хостлисты**, список `proxy`.
3. Включите обход и **в Telegram подключайтесь именно через этот прокси**.

> ℹ️ Пресеты не «чинят» сам прокси — они помогают до него дозвониться. Если ни один не пробивает ваш прокси —
> смените прокси-сервер, либо используйте **[tg-ws-proxy](https://github.com/Flowseal/tg-ws-proxy)** или
> **Cloudflare WARP**.

## Почему zapret2 (winws2), а не обычный zapret (winws/nfqws1)

Эта оболочка намеренно работает с **zapret2** — новым поколением проекта bol-van. Коротко, чем он
принципиально лучше предшественника (nfqws1 / winws первой версии):

### 1. Стратегии пишутся на Lua, а не зашиты в C
В **nfqws1** все техники «дурения» DPI (`fake`, `split`, `disorder`, `fooling` и т.д.) захардкожены в
C‑коде. Чтобы добавить новую технику или подправить существующую, нужно **писать C и пересобирать
бинарник** — высокий порог входа и медленная итерация.

В **nfqws2** C‑ядро отдаёт в **Lua** структурированное дерево пакета — «диссект», как в Wireshark — плюс
библиотеку хелперов (отправка пакетов, работа с бинарными данными, разбор TLS, поиск маркер‑позиций).
Стратегию теперь может написать или поправить **любой, кто понимает сети**, без компиляции. Это
превращает закрытый инструмент в открытую платформу.

### 2. Гибкость вместо «монстро‑комбайна» из сотен опций
nfqws1 со временем оброс сотнями частных флагов (`--dpi-desync-fake-tls`, `--dpi-desync-fake-http`,
`--dpi-desync-fake-quic`, `--dpi-desync-split-pos`, `--dpi-desync-fooling`, …). Это перегружало программу
и плохо масштабировалось: противостояние с регулятором требует всё более тонких и **меняющихся со
временем** воздействий, а старые перестают работать и забываются.

nfqws2 заменяет это **композицией Lua‑функций** (инстансов) и **блобами**. Нет жёстко зашитых фаз вроде
`fake,multisplit` — это просто последовательность вызываемых Lua‑функций, сколько нужно, каждая со своими
параметрами; одну и ту же можно вызвать несколько раз по‑разному.

### 3. Блобы вместо десятков «fake‑*» параметров
Вместо `--dpi-desync-fake-tls/http/quic` в nfqws2 есть **блобы** — произвольные бинарные данные (hex‑строкой
прямо в параметре или из файла). Стандартные фейки `fake_default_tls/http/quic` инициализируются
автоматически, а модификации (рандомизация, подмена SNI, дублирование session id) делаются одной строкой Lua.

### 4. Точечная фильтрация по типу пейлоада
Введено понятие **payload type** (`tls_client_hello`, `http_req`, `quic_initial`, `wireguard_initiation`,
`stun`, `dns_query` …). Стратегии применяются ровно к нужным пакетам — точнее и экономичнее по CPU, что
важно на слабых роутерах.

### 5. Полноценная работа в обе стороны и серверный режим
nfqws2 одинаково работает и с исходящими, и с входящими пакетами (`--out-range` / `--in-range`), есть режим
`--server`. Это открывает не только обход DPI, но и **обфускацию протоколов** (например, туннелирование
WireGuard‑UDP внутри ICMP‑пингов, XOR полезной нагрузки общим секретом).

### 6. Автоматическая TCP‑сегментация
Больше не нужно вручную считать размеры пакетов. Библиотека `zapret-lib.lua` отслеживает MSS соединения и
**сама сегментирует** то, что не влезает (например, большие TLS ClientHello с Kyber или большой `seqovl`).
В nfqws1 такое приводило к ошибке.

### Честная оговорка
zapret2 — **более мощный и гибкий**, но и **более сложный** инструмент; автор прямо пишет, что это «не
готовое решение для чайников». Базовое использование (готовые стратегии) — простое, и именно это берёт на
себя данная оболочка. Написание собственных стратегий требует понимания сетей. Для эволюционирующей борьбы
с DPI гибкость zapret2 — решающее преимущество.

## Требования

- Windows 10/11 x64.
- **Права администратора** — обязательны: WinDivert загружает драйвер в ядро (при запуске будет запрос UAC).
- Интернет при первом запуске (для загрузки движка).

```
publish\ZapretUI.exe
```
Self‑contained single‑file: .NET ставить не нужно. При первом запуске движок скачается, проверится по
SHA‑256 и распакуется в `%LOCALAPPDATA%\ZapretUI\engine`. Дальше: выбрать пресет → **Запустить**.

## Сборка из исходников

```powershell
# запуск для разработки
dotnet run --project ZapretUI.csproj

# релизный self-contained single-file exe (ничего не требует у пользователя)
dotnet publish ZapretUI.csproj -c Release -o publish
```

Для маленького бинарника (нужен установленный **.NET 9 Desktop Runtime**) соберите framework‑dependent:
уберите из `.csproj` свойства `SelfContained`, `PublishSingleFile`, `RuntimeIdentifier`.

## Где хранятся данные

Всё под `%LOCALAPPDATA%\ZapretUI\`:

| Папка/файл | Назначение |
|---|---|
| `engine\` | движок (winws2.exe, WinDivert, lua\, files\, windivert.filter\) + `installed_version.txt` |
| `lists\*.txt` | хостлисты (`youtube`/`discord`/`telegram`/`exclude`/`general`/`proxy` …) + IP-списки `ipset-*.txt` |
| `logs\` | логи движка и `fatal.log` |
| `presets.json` | пользовательские пресеты |
| `settings.json` | настройки приложения |

## Структура кода

- `Services/`:
  - `UpdaterService` — загрузка/проверка SHA‑256/распаковка движка с GitHub.
  - `EngineService` — запуск/остановка winws2, сборка аргументов (токены
    `{FILES}`/`{WF}`/`{HOSTLIST}` и именованные `{HOSTLIST:name}`/`{EXCLUDE:name}`/`{IPSET:name}`), логи.
  - `PresetService`, `HostlistService` (списки пере-засеваются из кода при запуске), `SettingsService`,
    `AutostartService`, `TargetService` (свои цели), `AppPaths`.
  - `ComboStrategyCatalog` + `AutoSelectService` — пул стратегий и автоподбор; `StrategyGeneratorService` —
    генерация персональной стратегии по сервисам.
  - `DiagnosticsService` + `NetProbe` — матрица доступности (ping / HTTP / TLS 1.2 / TLS 1.3).
  - `IpsetService` — IP‑список Discord (`mdig` → `ip2net`) и резолв хоста MTProxy.
  - `MonitorService` — фоновый сторож авто‑починки.
- `ViewModels/MainViewModel.cs` — координатор (MVVM).
- `Mvvm/` — `ObservableObject`, `RelayCommand`, конвертеры. `Themes/Theme.xaml` — тёмная тема.
- `MainWindow.xaml` — окно: кастомный заголовок, два режима, верхние вкладки
  (Главная · Стратегии · Хостлисты · Диагностика · Журнал · Настройки) + модальные окна.

## Замечания по безопасности

- Загрузка по HTTPS с GitHub, бинарники сверяются с `sha256sum.txt` из того же релиза.
- Из внешних зависимостей — только `Emoji.Wpf` (цветные эмодзи); остальное базовый .NET, WinForms‑трей и
  `schtasks.exe` для автозапуска.
- Распаковка архива защищена от path traversal.
